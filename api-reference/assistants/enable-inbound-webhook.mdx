---
title: "Ativar Webhook de Entrada do Assistente"
api: "POST /user/assistants/enable-inbound-webhook"
description: "Ative as notificações de webhook de entrada para um assistente específico"
---

Este endpoint ativa as notificações de webhook de entrada para um assistente, permitindo que você receba atualizações em tempo real sobre a conclusão de chamadas recebidas e seus dados.

### **Corpo da Requisição**

<ParamField body="assistant_id" type="integer" required>
  O ID do assistente para o qual o webhook será ativado
</ParamField>

<ParamField body="webhook_url" type="string" required>
  A URL para onde as notificações do webhook serão enviadas
</ParamField>

### Respostas

<ResponseField name="message" type="string">
  Mensagem de sucesso confirmando que o webhook foi habilitado
</ResponseField>

<ResponseField name="data" type="array">
  Array vazio (reservado para uso futuro)
</ResponseField>

### Respostas de Erro

<ResponseField name="404 Not Found">
  <Expandable title="Respostas de Erro">
    <ResponseField name="message" type="string">
      Mensagem de erro quando o assistente não é encontrado ou não pertence ao usuário autenticado
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="422 Validation Error">
  <Expandable title="Respostas de Erro">
    <ResponseField name="message" type="string">
      Mensagem de erro indicando falha na validação
    </ResponseField>
    <ResponseField name="errors" type="object">
      Erros detalhados de validação para cada campo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Success Response
{
  "message": "Webhook enabled successfully",
  "data": []
}
```


```json 404 Not Found
{
  "message": "Assistant not found"
}
```


```json 422 Validation Error
{
  "message": "The given data was invalid.",
  "errors": {
    "assistant_id": [
      "The assistant id field is required."
    ],
    "webhook_url": [
      "The webhook url field is required."
    ]
  }
}
```

</ResponseExample>
