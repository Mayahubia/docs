Este guia ajuda a resolver problemas comuns de integração SIP ao conectar seu provedor SIP à [**Mayahub.ai**](http://Mayahub.ai).

## Estratégia de Teste

- **Teste chamadas de saída primeiro** — são mais fáceis de configurar.
- **Teste chamadas de entrada depois** — após confirmar que as saídas funcionam.

---

### Problemas de SIP de Saída

**Problemas de Conexão**\
Tronco SIP não conecta para chamadas de saída:

- Verifique **credenciais SIP** — nome de usuário, senha e detalhes do servidor.
- **Tipo de telefone:** confirme com o provedor o tipo correto.
- **Configuração de IP fixo:** geralmente é necessário habilitar e inserir o IP do provedor.
- **Configurações do provedor:** verifique codec, transporte e portas.

---

### Problemas de Registro SIP

**Importante:** A plataforma **Mayahub.ai**:

- **Não envia** mensagens SIP REGISTER ao seu provedor.
- **Não requer** receber SIP REGISTER do seu provedor.
- Utiliza **autenticação direta de chamadas**.

---

### Testando Chamadas de Saída

- Configure as **definições do tronco SIP**.
- Faça uma **chamada de teste direta** pela plataforma.
- Verifique os **logs de chamadas** para identificar erros.

---

### Problemas de SIP de Entrada

**Configuração de Autenticação**\
Para chamadas de entrada:

- Use **autenticação por IP** (em vez de usuário/senha).
- **Liste os IPs do provedor (whitelist)** — adicione **todos** na plataforma.
- **Configuração do provedor:**
  - Use o **domínio FQDN** para roteamento (**não o IP de saída**).
  - Envie chamadas para o **domínio** — garante roteamento correto.
  - **Não envie para IP** — isso causa falhas de roteamento.

---

### Problemas Comuns de Entrada

Chamadas não chegam ao assistente:

- Verifique se **todos os IPs do provedor estão na whitelist**.
- Certifique-se de que o provedor **envia para o domínio FQDN** (não para o IP).
- Confirme o **formato do número de destino**.

---

### Problemas Gerais

**Rede:**

- Verifique a **conectividade da internet**.
- Confirme se não há **firewall bloqueando o tráfego SIP**.

**Compatibilidade do provedor:**

- Confirme que o provedor **suporta chamadas diretas** (sem registro).
- Verifique se há **suporte para autenticação por IP**.

---

### Checklist de Testes

**Antes de Contatar o Suporte**

**Teste de Saída:**

- Credenciais verificadas
- Tipo de telefone selecionado corretamente
- IP fixo configurado com o IP do provedor
- Chamada de teste direta realizada

**Teste de Entrada:**

- Autenticação por IP configurada
- Todos os IPs do provedor adicionados à whitelist
- Provedor configurado para enviar ao domínio FQDN (não IP direto)
- Chamada de teste feita de um número externo

**Geral:**

- Conectividade de rede verificada

### FAQ — Transferência de Chamada

**P:** Por que a função de transferência de chamadas não está funcionando?\
**R:** Se as transferências estiverem falhando, tente as seguintes soluções:

**Verifique as Configurações de Transferência SIP:**

- Defina o modo de transferência como **“Avançado”**.
- Use o formato SIP URI:

  ```
  sip:número@servidor-sip
  
  ```

  **Exemplo:** `sip:+5511999999999@sip.provedor.com:5060`

**Verifique o Suporte do Provedor:**

- Confirme que o provedor **suporta SIP REFER**.
- Confira se a funcionalidade de **transferência** está habilitada na sua conta.
- Alguns provedores exigem configuração específica para transferências.

**Teste Diferentes Formatos:**

- Com porta: `sip:+1234567890@sip-server:5060`
- Sem porta: `sip:+1234567890@sip-server`
- Simples: `sip:+1234567890`

**Verifique o Destino:**

- Certifique-se de que o número de destino **está ativo e recebendo chamadas normalmente**.
- Confirme que o número **não está bloqueado ou restrito**.

**Causas Comuns:**

- Provedor não suporta **SIP REFER**
- Formato **SIP URI incorreto**
- Bloqueio de rede/firewall em requisições de transferência
- Número de destino não atende

**Etapas de Solução:**

- Teste com um número conhecido que funcione
- Tente diferentes formatos SIP URI
- Contate seu provedor SIP para confirmar suporte à transferência
- Verifique os **logs de chamadas** para mensagens de erro específicas
