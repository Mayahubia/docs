O **system prompt** é o componente mais crucial do seu assistente de IA. Ele define a personalidade, comportamento, conhecimentos e capacidades do assistente. Pense nele como o **“cérebro” e o “manual de treinamento”** combinados.

## Início Rápido com Templates

Para começar rapidamente:

- Acesse as **configurações do assistente**
- Encontre o campo **system prompt**
- Clique no botão **Templates** próximo
- Escolha um template que corresponda ao seu caso de uso
- Personalize conforme suas necessidades

---

### **Suporte a Idiomas**

O system prompt pode ser escrito em qualquer idioma, independentemente da configuração de idioma falado:

- Escreva o prompt no idioma de sua preferência
- Defina o idioma falado separadamente nas configurações do assistente
- A IA seguirá as instruções do prompt enquanto fala no idioma selecionado

### **Exemplos:**

- System prompt em inglês, idioma falado definido para espanhol
- System prompt em alemão, idioma falado definido para francês
- System prompt em chinês, idioma falado definido para inglês

---

### **Por Que Eles Importam**

O system prompt:

- Define como a IA pensa e responde
- Estabelece limites da conversa
- Fornece conhecimentos essenciais
- Controla comportamento e tom
- Determina como lidar com casos excepcionais

---

### **Componentes Principais**

Um bom system prompt deve incluir:

### 1. Função & Identidade

```
Você é um representante de vendas profissional da [Empresa]. Você é especializado em [Produto/Serviço] e possui amplo conhecimento sobre nossos produtos e serviços.
```

### 2. Estilo de Conversação

```
Mantenha um tom amigável e profissional. Use linguagem clara e concisa. Evite jargões técnicos, a menos que sejam solicitados especificamente.
```

### 3. Informações Principais

```
Nossos principais produtos são:

* **Produto A ($X/mês):** [recursos]
* **Produto B ($Y/mês):** [recursos]
  **Promoção atual:** 20% de desconto nos primeiros 3 meses
```

### 4. Diretrizes de Comportamento

```
* Sempre verifique as informações do cliente antes de discutir detalhes da conta
* Transfira para um agente humano se o cliente parecer frustrado
* Não faça promessas sobre datas de entrega
```

### 5. Estrutura de Resposta

```
Ao ser questionado sobre preços:

1. Primeiro, compreenda as necessidades do cliente
2. Apresente as opções de pacotes relevantes
3. Explique a proposta de valor
4. Informe quaisquer descontos aplicáveis
```

## Melhores Práticas

### Seja Específico

- Instruções claras geram melhores resultados
- Inclua exemplos de boas respostas
- Defina o que **NÃO** deve ser feito

### A Estrutura Importa

- Organize as informações de forma lógica
- Use listas e seções
- Mantenha informações relacionadas juntas

### Testar e Iterar

- Comece com um prompt básico
- Teste diversos cenários
- Refine com base em gravações de chamadas
- Adicione instruções para casos excepcionais

### Erros Comuns

- **Muito vago:** "Seja útil e profissional" (não é específico suficiente)
- **Muito rígido:** Roteirizar todas as respostas possíveis (reduz a naturalidade)
- **Excesso de informação:** Incluir detalhes desnecessários
- **Diretrizes ausentes:** Não especificar como lidar com situações comuns

## Exemplo de Estrutura

```
# Função e Propósito

[Defina quem é a IA e seus principais objetivos]

# Conhecimentos Essenciais

[Informações essenciais sobre produtos/serviços]

# Diretrizes de Conversa

[Como interagir com os clientes]

# Padrões de Resposta

[Como lidar com situações específicas]

# Limitações e Restrições

[O que a IA **não deve** fazer]
```

## Testando Seu Prompt

Faça chamadas de teste cobrindo:

- Cenários comuns
- Casos excepcionais
- Situações difíceis
- Diferentes perfis de clientes

### Revise e ajuste:

- Ouça as gravações das chamadas
- Verifique a precisão das respostas
- Confirme a consistência do tom
- Teste a retenção de conhecimento

### **Dicas de Otimização**

- Comece com o **Fast Engine** para iterações rápidas
- Use gravações de chamadas para identificar lacunas
- Adicione exemplos de respostas boas/ruins
- Inclua instruções para perguntas inesperadas

---

**Dica Profissional:** Seu system prompt é um documento vivo — atualize-o regularmente com base nas experiências reais de chamadas e nas interações com os clientes.
