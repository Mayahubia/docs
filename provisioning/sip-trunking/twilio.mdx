## **Etapa 3: Importar números para a Mayahub**

Agora que o número está configurado com o seu **Elastic SIP Trunking**, é necessário **importar o número para a Mayahub** para que o sistema saiba como rotear as chamadas.

1. Acesse o painel da [**Mayahub.ai**](http://Mayahub.ai)
2. Vá até **Seus Números de Telefone**
3. Clique em **“Integrar tronco SIP”**

<Frame>
  ![Import numbers to Autocalls](/resources/twillio-import-autocalls-sip1.png)
</Frame>

4. Preencha suas informações com base no tipo de integração que você escolheu.

```
Número de telefone: exemplo: +1 555 0123
Nome de usuário: nome de usuário da sua extensão
Senha: senha da sua extensão
```

<Frame>
  ![Fill in your details based on your integration type](/resources/twillio-autocalls-auth-integration.png)
</Frame>

5. Aqui você deve inserir o **URI SIP de terminação** que configurou na **Etapa 1**.

```
Endereço do servidor: exemplo: xxx.pstn.twilio.com
```

<Frame>
  ![Twillio Connect Number via SIP](/resources/twillio-outbound-settings.PNG)
</Frame>

6. Aqui, selecione o **tipo de autenticação** conforme a opção escolhida na **Etapa 2**.

   Se você selecionou o **tipo IP**, adicione os **endereços IP** correspondentes.

```
54.244.51.0
  54.244.51.1  
  54.244.51.2
  54.244.51.3
54.172.60.0
  54.172.60.1
  54.172.60.2
  54.172.60.3
```

<Frame>
  ![Twillio Autocalls Credentials IP Type](/resources/twillio-autocalls-credentials-ip-type1.png)
</Frame>

7. Selecione o país onde o seu tronco SIP está localizado.

   <Frame>
     ![Twillio Autocalls SIP Country](/resources/twillio-autocalls-sip-country.png)
   </Frame>

### **Etapa 4: Verificação da Configuração**

Após concluir a configuração:

1. Faça uma **chamada de teste** para o seu número ou extensão.
2. Verifique se o **assistente de IA atende corretamente**.
3. Use o **recurso de teste de chamada** para confirmar se as **chamadas de saída** estão funcionando.

## **Requisitos de Segurança**

Suas credenciais SIP precisam estar seguras:

- **Nome de usuário:** seu número de extensão ou o nome de usuário fornecido pelo provedor.
- **Senha** deve conter:
  - 12 ou mais caracteres
  - Números
  - Letras maiúsculas e minúsculas
  - Caracteres especiais (recomendado)
