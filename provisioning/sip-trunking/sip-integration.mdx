## O que é Integração SIP?

A integração **SIP (Session Initiation Protocol)** permite conectar seu sistema telefônico existente à [**Mayahub.ai**](http://Mayahub.ai).\
Pense nisso como uma ponte entre sua configuração telefônica atual e nossa plataforma de IA.\
Isso significa que você pode:

- Usar seus **números de telefone existentes** com nossos assistentes de IA
- **Manter** seu sistema telefônico e provedor atuais
- **Fazer e receber chamadas** através da sua infraestrutura existente
- **Manter controle total** sobre seus números
- **Reduzir custos de operadora** ao usar provedores VoIP locais com tarifas mais competitivas

---

## Como Funciona

Ao integrar seu sistema telefônico com a [**Mayahub.ai**](http://Mayahub.ai):

- **Chamadas de Entrada:** As chamadas para seus números são roteadas através do seu sistema telefônico até nossos assistentes de IA.
- **Chamadas de Saída:** Nossos assistentes de IA podem realizar chamadas exibindo o seu número.
- **Gerenciamento:** Você continua administrando seus números através do seu provedor atual.
- **Eficiência de Custos:** Aproveite as tarifas competitivas do seu provedor VoIP para terminação de chamadas.

---

## Opções de Integração

Você pode conectar seu sistema telefônico à [**Mayahub.ai**](http://Mayahub.ai) de duas formas:

---

## 1. Integração por Extensão PBX

É como conceder à nossa plataforma de IA uma **“extensão”** dentro do seu sistema telefônico:

**O que é:**\
Conecta uma **extensão específica** do seu PBX/sistema telefônico.

**Ideal para:**

- Testar a integração
- Roteamento de extensões específicas para a IA
- Configuração simples com sistemas telefônicos já existentes

**Você vai precisar de:**

- Seu **número de extensão** (exemplo: 1000)
- **Nome de usuário e senha** da extensão
- **Endereço do servidor PBX** (exemplo: [pbx.suaempresa.com](http://pbx.suaempresa.com))

## Exemplo prático:

```
Seu escritório usa as extensões **1001–1010** para a equipe.
Você configura a extensão **1011** para o assistente de IA.
As chamadas para **1011** são direcionadas para a IA.
A IA pode fazer chamadas exibindo o número da extensão **1011**.
```

## 2. Integração Direta de Número

Essa opção conecta **números de telefone específicos** diretamente à nossa plataforma:

**O que é:**\
Roteia um ou mais dos seus números de telefone para o nosso assistente de IA.

**Ideal para:**

- Linhas telefônicas dedicadas à IA
- Números de atendimento ao cliente
- Linhas de vendas ou suporte

**Você vai precisar de:**

- Seu **número de telefone** (exemplo: +55 11 5550-1234)
- **Credenciais SIP** fornecidas pelo seu provedor
- **Endereço do servidor SIP** do provedor

**Exemplo prático:**

```
Você tem um número de suporte: +1 555 0123
Você conecta esse número à nossa IA.
Todas as chamadas para+1 555 0123 são atendidas pelo assistente de IA.
A IA realiza chamadas de saída exibindo o número 1 3555 012.
```

## **Autenticação e Fluxo de Chamadas**

**Configuração Bidirecional**

A integração SIP usa as **mesmas credenciais** para chamadas de entrada e de saída:

- [******Chamadas de Entrada (VoIP → Mayahub.ai):******](http://Mayahub.ai)

  ```
  Seu sistema VoIP se autentica usando nome de usuário e senha
  ↓
  As chamadas chegam à nossa plataforma
  ↓
  O assistente de IA atende
  
  ```
- **Chamadas de Saída (**[**Mayahub.ai**](http://Mayahub.ai)\*\* → VoIP):\*\*

  ```
  A IA inicia a chamada
  ↓
  Nossa plataforma se autentica no seu sistema VoIP usando o mesmo nome de usuário e senha
  ↓
  A chamada chega ao destino
  
  ```

### **Sincronização de Credenciais**

Para que a integração funcione corretamente:

1. As credenciais configuradas no \*\*painel da [**Mayahub.ai**](http://Mayahub.ai) devem corresponder **exatamente** à configuração do seu sistema VoIP.
2. A **mesma combinação de nome de usuário e senha** é usada para ambas as direções (entrada e saída).
3. Quaisquer **alterações nas credenciais** devem ser atualizadas em **ambos os sistemas**.

<Warning>
  Se você alterar as credenciais no seu sistema VoIP, lembre-se de atualizá-las também no painel da [**Mayahub.ai**](http://Mayahub.ai) (e vice-versa) para manter a conexão.
</Warning>

### **Etapa 2: Configure no Painel**

1. Acesse o painel da [**Mayahub.ai**](http://Mayahub.ai)
2. Vá até **Seus Números de Telefone**
3. Clique em **“Integrar tronco SIP”**

---

**Etapa 3: Selecione o Tipo de Tronco SIP**

Na parte superior do formulário, selecione um dos seguintes tipos de tronco:

- **Extensão SIP:** use esta opção se você já possui uma credencial SIP existente (uma extensão) fornecida pelo seu provedor VoIP.
- **Número de Telefone (DID):** use esta opção se desejar integrar um número **DID (Discagem Direta)** dedicado em vez de uma extensão SIP genérica.

---

### **Etapa 4: Configurações de Saída**

Essas configurações controlam como a [**Mayahub.ai**](http://Mayahub.ai) envia chamadas de saída através do seu provedor SIP.

- **Endereço SIP:** insira o endereço/servidor do tronco SIP **sem a porta**.
- **Proxy de Saída (IP Fixo):** alguns provedores SIP exigem que as chamadas se originem de um único endereço IP permitido (whitelist). Configure da seguinte forma:
  - **Ativar (Toggle ON):** se o seu provedor SIP exigir um **IP de origem fixo**, ative esta opção — o proxy de saída virá de um **endereço IP estático conhecido**. A [**Mayahub.ai**](http://Mayahub.ai) enviará todas as solicitações de saída através desse IP fixo. Use essa opção **apenas se o seu provedor exigir explicitamente um IP de origem único**.
  - **Desativar (Toggle OFF):** se o seu provedor aceitar chamadas de **qualquer IP**, deixe esta opção desativada. Nesse caso, a [**Mayahub.ai**](http://Mayahub.ai) enviará chamadas a partir de **IPs dinâmicos**.

```
Endereço SIP: (exemplo: pbx.empresa.com / sip.provedor.com)
```

### **Etapa 5: Configuração do Formato de Número**

Esta etapa informa ao nosso sistema como formatar os números de telefone ao realizar chamadas.\
Escolha a opção de acordo com os requisitos do seu provedor:

1. **Internacional com +**

   ```
   Formato: +[código do país][número]
   Exemplo: +1 202 555 0123
   Requisição SIP: sip:[+12025550123@seu.pbx.com](mailto:+12025550123@seu.pbx.com)
   
   ```
2. **Internacional sem +**

   ```
   Formato: [código do país][número]
   Exemplo: 1 202 555 0123
   Requisição SIP: sip:[12025550123@seu.pbx.com](mailto:12025550123@seu.pbx.com)
   
   ```
3. **Formato Nacional**

   ```
   Formato: [número local]
   Exemplo: 0741926265
   Requisição SIP: sip:[0741926265@seu.pbx.com](mailto:0741926265@seu.pbx.com)
   
   ```

### Etapa 6: Configurações de Entrada

Essas configurações determinam como a **Mayahub.ai** recebe e autentica chamadas de entrada do seu provedor SIP.

1. **Nosso Endereço SIP:**\
   A **Mayahub.ai** atribui um URI SIP exclusivo para entrada: `5wed79aeavr.sip.livekit.cloud`.\
   Seu provedor SIP deve ser configurado para enviar as chamadas de entrada para esse endereço, direcionando o tronco de saída ou a regra de roteamento para esse URI.
2. **Endereço IP (Lista de IPs Permitidos):**\
   Se o seu provedor SIP puder enviar chamadas a partir de um conjunto conhecido de endereços IP públicos, escolha **Endereço IP**.\
   Em seguida, adicione cada IP do provedor à **lista de permissões (whitelist)** em **Autenticação de Entrada**.
3. **Nome de Usuário e Senha:**\
   Se o seu provedor oferecer suporte à autenticação SIP, escolha **Nome de usuário e senha**.\
   As mesmas credenciais inseridas nas **Configurações de Saída (username/password)** serão usadas para autenticar as chamadas de entrada.\
   Certifique-se de que o PBX do seu provedor esteja configurado para incluir essas credenciais.

### **Etapa 8: Verificação da Configuração**

Após concluir a configuração:

1. Faça uma **chamada de teste** para o seu número ou extensão.
2. Verifique se o **assistente de IA atende** corretamente.
3. Use o **recurso de teste de chamada** para confirmar se as chamadas de saída estão funcionando.

---

**Requisitos de Segurança**

Suas credenciais SIP precisam estar seguras:

- **Nome de usuário:** seu número de extensão ou o nome de usuário fornecido pelo provedor.
- **Senha** deve conter:
  - 12 ou mais caracteres
  - Números
  - Letras maiúsculas e minúsculas
  - Caracteres especiais (recomendado)
