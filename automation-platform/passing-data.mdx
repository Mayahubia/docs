### **Fluxo de Dados**

Todo fluxo é um diagrama vertical que começa com uma etapa de gatilho (_trigger_), seguida por qualquer número de etapas de ação.

As etapas são conectadas verticalmente. Os dados fluem das etapas “pai” para as etapas “filhas”. As etapas “filhas” têm acesso aos dados gerados pelas etapas anteriores.

### **Exemplo de Etapas**

<video width="450" autoPlay muted loop playsinline src="/resources/passing-data-3steps.mp4" />

Este fluxo tem 3 etapas, e o acesso aos dados funciona assim:

- **Etapa 1**: é a principal produtora de dados que serão usados nas próximas etapas. Os dados produzidos por ela estarão acessíveis nas Etapas 2 e 3. Alguns gatilhos, como os de agendamento (_Schedules_), não produzem dados.
- **Etapa 2**: pode acessar os dados da Etapa 1. Após sua execução, ela também gera dados para as próximas etapas.
- **Etapa 3**: pode acessar os dados das Etapas 1 e 2, pois são suas etapas anteriores. Ela também pode gerar dados, mas como é a última do fluxo, esses dados não serão usados por nenhuma outra etapa.

## **Painel "Inserir Dados"**

Para usar os dados de uma etapa anterior na etapa atual, basta posicionar o cursor em qualquer campo de entrada. O painel "Inserir Dados" aparecerá automaticamente.

<video autoPlay muted loop playsinline src="/resources/passing-data-data-to-insert-panel.mp4" />

Esse painel mostra as etapas acessíveis e seus respectivos dados. Você pode expandir os itens para ver seu conteúdo e clicar sobre eles para inseri-los no campo desejado.

Se um item tiver um ícone de seta (⌄) à direita, significa que ele pode ser expandido para exibir propriedades internas. Você pode inserir o item completo ou uma de suas propriedades.

Quando você insere um dado, ele será adicionado exatamente na posição do cursor. Isso permite combinar texto fixo com dados dinâmicos em qualquer campo.

<video autoPlay muted loop playsinline src="/resources/passing-data-main-insert-data-example.mp4" />

**Recomendação:** sempre expanda os itens antes de inseri-los para verificar se os dados são compatíveis com o campo que você está preenchendo.

## **Testando Etapas para Gerar Dados**

É necessário testar uma etapa antes de acessar seus dados. Isso evita erros e garante que o fluxo funcione corretamente após a publicação.

Se você tentar acessar dados de uma etapa não testada, verá a mensagem:

<img
  width="350"
  src="/resources/passing-data-test-step-first.png"
  alt="Test your automation step first"
/>

Para resolver isso, vá até a etapa e utilize o painel _Generate Sample Data_ para testá-la. Existem formas diferentes de teste:

- **Load Data** (_Carregar Dados_): alguns gatilhos permitem carregar dados da conta conectada automaticamente.
- **Test Trigger** (_Testar Gatilho_): exige que você acione o gatilho manualmente na conta conectada para gerar dados.
- **Send Data** (_Enviar Dados_): webhooks exigem que você envie uma requisição de teste para gerar os dados.
- **Test Action** (_Testar Ação_): permite executar a ação para gerar dados de amostra.

### **Acessando dados por caminho**\
Se você não encontrar os dados desejados no painel "Inserir Dados", é possível escrever o caminho JSON manualmente.

Use a seguinte sintaxe para isso:

`{{slug_da_etapa.caminho.para.propriedade}}`

O `slug_da_etapa` pode ser visualizado ao passar o cursor sobre qualquer etapa do seu fluxo — ele aparecerá à direita da etapa.
